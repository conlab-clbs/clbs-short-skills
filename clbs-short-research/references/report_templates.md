## 02_hook_teardown_<platform>.md（媒体別・心臓部）

```markdown
# 02. 冒頭フック解剖 — [媒体名]

> 取得日: YYYY-MM-DD ／ 素材: 自動取得 research/clips/<platform>/（YouTube/TikTok）または ユーザー提供スクショ（Instagram）

## 異常値 #1: 「[タイトル/キャプション]」
- 再生数: XX万 ／ いいね: XX ／ 投稿日: ... ／ フォロワー: ...（[要確認]可）／ 尺: XX秒
- URL: ...

### ★冒頭1〜2秒フック（最重要）
- **何が写っているか**: （0.0s〜2.0sのフレームを順に。被写体・構図・動き）
- **焼き込みテロップ**: 「（フレーム上の文字をそのまま）」
- **何を喋っているか**: 「（冒頭セリフ）」
- **フック型**: ① 常識破壊 ／ 常識破壊フレーム: 7. 落とし穴
- **視覚×言語の分担**: テロップで結論を断言、音声で問いを投げる、など
- **権威ソース**: 「○○大学の報告」等の有無

### 興味づけ〜本題〜CTA（簡潔に）
- 興味づけ: 「やらないと/やれば〜保存して何度も見て」型の有無
- 本題の見せ方: 図/表/映像素材/会話調 のどれで深さを出すか
- CTA: コメント誘導 / プロフィール誘導 / フォロー誘導

### この異常値の効いている理由（1〜2行）
> ...

## 異常値 #2: ...
（同形式で並べる）

## [媒体名] の傾向まとめ
- この媒体でのフックの効き方の共通点
- テロップの長さ・出し方の癖
- 真似できるポイント / 真似しない方がいいポイント
```

    hook_type: ""           # ①常識破壊 / ②ネガティブ / ③短期快楽 / ④質問 / ⑤コレをやると◯◯

    visual: ""              # 冒頭に何が写っているか
    telop: ""               # 焼き込みテロップ（そのまま）

      source_platform: ""
      imitate: ""
  one_line_formula: ""      # バズ方程式（1行）

# 後段への引き渡しメッセージ
handoff_to_script_skill: |

  - hook_library の効いた型を冒頭2秒フックの起点にする（順算思考）
