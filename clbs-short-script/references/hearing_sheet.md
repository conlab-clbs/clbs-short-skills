# ヒアリングシート（Phase 0）

> 前段 `clbs-short-research` をやっていれば `04_short_research_summary.yaml` を渡してもらう。
> それがあれば 1〜4・8 はYAMLから埋まる。無ければ以下を聞く（不明は「不明」でOK・こちらで提案）。

```
ショート動画の台本を作っていきましょう。

（リサーチ済みなら 04_short_research_summary.yaml を渡してください。なければ以下に回答を）

1. クライアントのジャンル / バックエンド商品
   例：50代起業／ボディメイク／金運メソッド／引き寄せ など
2. 想定視聴者（ペルソナ）の属性・「不」
   不安／不満／不足／不快／不信 のどれを抱えているか
3. 破壊する常識（視聴者が信じている思い込み）
4. 引用したい権威ソース（任意：研究／大学名／統計／実績／ニュース）
5. 投稿先（複数可）
   □ TikTok  □ YouTube Shorts  □ Instagram Reels
6. CTA方針
   □ コメント誘導（認知拡大期・バズ狙い）
   □ プロフィール誘導（リスト獲得期・無料オファーへ）
7. 想定尺
   □ 30秒  □ 60秒（推奨）  □ 90秒
8. マス〜ニッチのどこを狙うか
   □ マス（エンタメ寄り・広く）  □ ミドル（ノウハウ）  □ ニッチ（バックエンド寄り）
   ※ 初期は マス:ニッチ=2:1、伸びてきたら 1:2 推奨
9. 顔出しの有無 / 既存の素材（あれば）
```

## 受領後の確認（Phase 1）
回答を短く整理して提示し、研究サマリがあれば hook_library / models_to_inherit を要約。
「この内容で企画5案を出します」と宣言して Phase 2 へ。
